## III. Demo Script

### Standard Demo Structure (30-Minute Version)

```
Time     Segment              Content                                   Goal
0-2min   Opening & Rapport    Self-intro, agenda, understand customer   Build rapport
2-5min   Customer Pain Point  Echo customer pain points, build trust   Get customer confirmation
         Resonance
5-15min  Core Workflow Demo   Demo core value scenarios (real data)    Show customer the value
15-22min Differentiated       Demo unique features competitors lack    Create a memorable moment
         Highlights
22-27min Customer Scenario    Tailored demo for customer's industry/   Help customer envision
         Mapping              scenario                                  themselves using it
27-30min Q&A + Next Steps     Answer questions, clarify follow-ups     Advance the opportunity
```

### Demo Scenario Design Principles

1. **Use the customer's data, tell the customer's story**
   - Research the customer's industry beforehand; use that industry's data and terminology
   - The more realistic the scenario, the stronger the impact

2. **A demo is not a feature list; it's a value demonstration**
   - Don't click through menus one by one: "This is our list page, this is our detail page..."
   - Instead: "What you're seeing now is Manager XX's dashboard today — in 3 seconds they know which purchase orders need attention..."

3. **Only demo 3 core scenarios — don't overdo it**
   - B2B products have many features; demoing everything will overwhelm the customer
   - Choose the 3 scenarios most likely to resonate with the customer's role

4. **Prepare backup pages**
   - If the customer asks about a feature you didn't prepare, quickly jump to show it
   - Don't fumble around live; prepare bookmarks in advance

### Demo Environment Checklist

```
Before Demo:
□ Is the demo environment data refreshed? (Latest dates, realistic feel)
□ Are all features working? (Permissions/Approval workflows/Notifications)
□ Is the network stable? (Backup hotspot ready)
□ Is the screen resolution appropriate? (Customer projection screens are often low-res)
□ Any sensitive data exposed? (Clean real info from test data)
□ Need to switch roles during demo? (Prepare login accounts for different roles)
```

## V. ROI Calculator

### Design Principles
- Customer fills in minimal inputs (3-5 fields); system auto-calculates ROI
- Use the customer's own language and metrics
- Results visualized (charts + numbers)

### ROI Calculation Logic

```
Customer Inputs:
├── Current monthly labor hours for [task]: [X] hours
├── Current monthly loss due to [problem]: [¥X]
├── Current monthly labor cost per employee: [¥X]
└── Expected number of employees using the product: [X] people

Auto Calculation:
├── Efficiency Gain = [X] hours × 70% (efficiency improvement rate) × [labor cost/hour]
├── Risk Avoidance = [monthly loss] × 80% (risk reduction rate)
├── Annual Total Savings = (Efficiency Gain + Risk Avoidance) × 12
├── Annual Product Cost = [license fees] + [implementation fees]
├── ROI = (Annual Total Savings - Annual Product Cost) / Annual Product Cost × 100%
└── Payback Period = Annual Product Cost / (Annual Total Savings / 12) months
```

## VI. Objection Handling Handbook

### Common Objections & Response Strategies

| Objection Type | What Customer Says | What They Really Mean | Response Strategy |
|---------|-----------|------------|---------|
| Price Objection | "Too expensive" | Haven't seen enough value | Don't discount — demonstrate more value. Use the ROI calculator |
| Feature Objection | "Competitor XX has this feature" | This feature is important to them | Confirm if truly needed; if yes → feed back to product; if no → educate |
| Trust Objection | "Never heard of you" | Worried about stability | Show customer case studies, certifications, funding/IPO info |
| Process Objection | "Our internal processes are very complex" | Worried about implementation difficulty | Provide implementation methodology, customer success stories |
| Competitor Objection | "We're already using XX" | Migration cost is high | Quantify migration cost, compare with long-term benefits |
| Timing Objection | "Let's wait and see" | Not a priority | Dig for the real reason — budget/Pain point not acute enough/Decision-maker absent? |

### LAER Model for Handling Objections

```
Listen: Let the customer finish speaking; don't interrupt; note their exact words
Acknowledge: Validate before guiding
  "I understand your concern — many customers initially had the same thought..."
Explore: Dig deeper into the underlying reasons
  "Aside from price, what other aspects are making you hesitate?"
Respond: Address the real reason with a targeted response
  "On this point, we had a customer in the exact same situation — here's how they resolved it..."
```---
